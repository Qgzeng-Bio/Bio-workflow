# Program onboarding evidence bundle schema

Third-version onboarding writes evidence under:

```text
reports/program-onboarding/<program_key>/<timestamp>/
```

The bundle is appendable across the onboarding sequence. A missing file means the
corresponding step has not run yet.

By default, helper subcommands reject evidence and draft paths outside the project
layout above. `--allow-external-paths` exists only for non-install smoke tests and
must not be used for installation.

## Files

- `intent.tsv`: program name, normalized key, action, status, timestamp, and
  optional blocking reason.
- `local_discovery.tsv`: cheap discovery records from `command -v`, explicit
  executable paths, Conda/Mamba env listing, and help/version attempts.
- `install_proposal.json`: machine-readable proposal generated by
  `plan-install`.
- `install_proposal.md`: review copy of the proposal.
- `install.log`: stdout/stderr and exit code from `install --yes`.
- `environment_proof.md`: executable path, Conda context, captured commands, and
  startup-check status.
- `version.txt`: captured version command and output.
- `help.txt`: captured help command and output.
- `command_attempts.tsv`: all help/version attempts, including failed attempts
  that are not usable evidence.
- `card_draft.md`: evidence-local copy of the generated draft program card.

`draft-card` also writes:

```text
references/program-cards/drafts/<program_key>.md
```

Draft cards are not active cards and must not be registered automatically.

## Discovery constraints

Allowed local probes are bounded and cheap:

- `command -v <program>`
- explicit user-provided executable path
- `<program> --version`, `version`, `-v`, or `-version`
- `<program> --help`, `-h`, or `help`
- `conda env list` or `mamba env list`

Do not scan `/data9`, `/data9/home/qgzeng`, or broad project roots to locate a
program. Ask for a bounded root, filename pattern, and maxdepth if a search is
truly needed.

## Evidence grades

Generated drafts may use at most:

- `local_help` when local help or version evidence exists
- `inferred` for mode, input, parameter, and resource claims that still need
  review

Failed help/version attempts are recorded in `command_attempts.tsv`, but they do
not create `help.txt` or `version.txt` and must not upgrade a draft to
`local_help`.

Do not write `local_run` until a real pilot or production run succeeds. Do not
write `project_history` unless a completed project run proves the claim.
